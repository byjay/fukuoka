# 후쿠오카 여행 웹사이트 개선 계획 (겨울 컨셉 & 모바일 최적화)

## 1. 핵심 요구사항 및 기술 전략

| 요구사항 | 현재 상태 (index.html 분석) | 개선 전략 |
| :--- | :--- | :--- |
| **겨울 컨셉 변경** | 🌸 벚꽃/봄 컨셉 (핑크, 밝은 색상) | ❄️ **겨울 컨셉** (딥 블루, 실버, 골드 강조)으로 색상 팔레트 및 이미지 전면 교체 |
| **모바일 완벽 대응** | Tailwind CSS를 사용하고 있으나, 메뉴 및 레이아웃 개선 필요 | Tailwind CSS를 활용한 **모바일 퍼스트** 반응형 디자인 강화 및 메뉴 UI/UX 개선 |
| **메뉴/UI 개선** | 상단 스크롤 메뉴 (Lousy Menu) | **화면 전환** 기반의 명확한 네비게이션 구조로 변경 (멀티 페이지 또는 SPA 라우팅) |
| **장소 정보 보강** | 10개 미만의 장소 정보 | 최소 **40개 이상의 장소**에 대한 상세 정보 수집 및 구조화 |
| **화면 전환** | 앵커 링크 기반 스크롤 이동 | **멀티 페이지 구조** (index.html, day1.html, day2.html 등) 또는 JavaScript 라우팅 도입 |
| **병렬 작업 & GitHub 업로드** | - | 콘텐츠 수집 및 웹 개발 작업을 병렬화하고, 최종 결과물을 `fukuoka_project` 저장소에 커밋 및 푸시 |

## 2. 겨울 컨셉 디자인 정의

*   **주 색상 (Primary):** 딥 블루 (#003366) - 겨울 밤하늘
*   **보조 색상 (Secondary):** 실버/화이트 (#E2E8F0, #FFFFFF) - 눈, 서리
*   **강조 색상 (Accent):** 골드 (#FFD700) - 크리스마스 조명, 온천의 따뜻함
*   **주요 이미지:** 하카타역 일루미네이션, 모츠나베, 온천 풍경.

## 3. 데이터 구조 정의 (locations.json)

모든 장소 정보는 JSON 파일로 구조화하여 관리하며, 이는 웹사이트 콘텐츠 업데이트의 기반이 됩니다.

```json
[
  {
    "id": "unique_id_for_location",
    "category": "food" | "attraction" | "shopping" | "transport",
    "sub_category": "ramen" | "motsunabe" | "shrine" | "park" | "etc",
    "name_ko": "장소 이름 (한국어)",
    "name_ja": "장소 이름 (일본어)",
    "name_en": "Location Name (English)",
    "address_ko": "주소 (한국어)",
    "address_ja": "주소 (일본어)",
    "latitude": 33.xxxx,
    "longitude": 130.xxxx,
    "hours": "영업 시간 (예: 10:00 - 22:00)",
    "price_range": "가격대 (예: 1000엔 ~ 3000엔)",
    "details_ko": "장소에 대한 상세 설명 (한국어)",
    "details_ja": "장소에 대한 상세 설명 (일본어)",
    "rating": 4.5,
    "review_count": 12345,
    "transportation": [
      {"type": "subway", "line": "공항선", "station": "하카타역", "exit": "동쪽 출구"},
      {"type": "bus", "route": "100엔 순환 버스", "stop": "캐널시티 앞"}
    ],
    "nearby_attractions": ["id_of_nearby_place_1", "id_of_nearby_place_2"],
    "images": [
      "url_to_image_1",
      "url_to_image_2",
      // ... 최소 10개
    ],
    "winter_tip_ko": "겨울철 방문 팁 (예: 따뜻한 모츠나베 추천)"
  }
]
```

## 4. 3박 4일 겨울 추천 일정 구조

사용자 요청에 따라, 확정된 3박 4일 일정과 함께 최소 40개의 옵션 장소를 제공합니다.

| 일차 | 시간대 | 확정 일정 (Fixed Itinerary) | 옵션 장소 (Optional Locations) |
| :--- | :--- | :--- | :--- |
| **1일차** | 오전/오후 | 후쿠오카 공항 도착, 호텔 체크인 (하카타) | 하카타역 주변 쇼핑 (아뮤 플라자, 한큐 백화점) |
| | 저녁 | **확정:** 하카타역 크리스마스 마켓 & 일루미네이션 | **확정:** 하카타 잇소우 라멘 또는 모츠나베 맛집 |
| **2일차** | 종일 | **확정:** 유후인/벳푸 일일 버스 투어 (다자이후, 유후인, 벳푸 지옥 온천) | - (투어 일정에 집중) |
| **3일차** | 오전/오후 | **확정:** 텐진 쇼핑 (파르코, 텐진 지하상가) 및 다이묘 거리 | 오호리 공원 산책, 후쿠오카 성터 |
| | 저녁 | **확정:** 나카스 야타이 거리 (겨울철 따뜻한 오뎅/꼬치) | 후쿠오카 타워 야경 (겨울 한정 라이트업) |
| **4일차** | 오전 | **확정:** 오호리 공원 산책 또는 쇼핑몰에서 기념품 구매 | 구시다 신사 (겨울 참배), 캐널시티 하카타 |
| | 오후 | 후쿠오카 공항으로 이동 및 귀국 | - |

## 5. 웹사이트 구조 변경 (화면 전환 구현)

스크롤 대신 화면 전환을 위해 웹사이트를 멀티 페이지 구조로 변경합니다.

| 파일명 | 역할 | 내용 |
| :--- | :--- | :--- |
| `index.html` | 메인 페이지 | 여행 개요, 겨울 컨셉 소개, 3박 4일 일정 요약 |
| `day1.html` | 1일차 상세 | 1일차 확정 일정 및 주변 옵션 장소 목록 |
| `day2.html` | 2일차 상세 | 2일차 확정 일정 및 주변 옵션 장소 목록 |
| `day3.html` | 3일차 상세 | 3일차 확정 일정 및 주변 옵션 장소 목록 |
| `day4.html` | 4일차 상세 | 4일차 확정 일정 및 주변 옵션 장소 목록 |
| `locations.html` | 전체 장소 목록 | 40개 이상의 모든 옵션 장소를 카테고리별로 검색/필터링 가능하게 구현 |
| `data/locations.json` | 데이터 파일 | 40개 이상의 장소 상세 정보 (위 3번 구조 참조) |

이 계획을 바탕으로 다음 단계인 **병렬 데이터 수집 및 콘텐츠 작성**을 진행하겠습니다.
